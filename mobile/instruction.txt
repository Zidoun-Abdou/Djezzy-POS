## Bug Fixes (2025-12-26) - Compilation Errors

### 6. Fixed Compilation Errors in read_nfc_page.dart

**Problem:**
After removing face matching feature, two compilation errors occurred:
1. `IDCardService.markAsVerified()` was called without required `confidence` parameter
2. `UserProfilePage` was missing required `userData` parameter

**Solution in `lib/idcard/read_nfc_page.dart`:**

```dart
// BEFORE (line 182):
await IDCardService.markAsVerified();

// AFTER:
await IDCardService.markAsVerified(1.0);  // 1.0 confidence for NFC verification

// BEFORE (line 193):
builder: (_) => UserProfilePage(cameras: widget.cameras),

// AFTER:
final userData = await IDCardService.getUserData();
if (mounted && userData != null) {
  Navigator.of(context).pushReplacement(
    MaterialPageRoute(
      builder: (_) => UserProfilePage(
        userData: userData,
        cameras: widget.cameras,
      ),
    ),
  );
}
```

**Notes:**
- Confidence is set to `1.0` since NFC chip data is considered fully verified
- User data is fetched from IDCardService after saving NFC data
- Navigation only occurs if userData is successfully retrieved

## PDF Contract Generation Fixes (2025-12-30)

### 13. Fixed PDF Generation Issues

**Problems Identified:**
1. Arabic characters showing as "XXXXX" boxes
2. Offer section empty (no name, price, phone, data, validity)
3. Signature image not rendering
4. Customer face photo missing from PDF

**Root Causes:**
- Default PDF font (Helvetica) doesn't support Arabic Unicode
- Offer section content was being cut off
- Face image was never included in PDF layout

**Solutions Implemented:**

#### 1. Arabic Font Support
- Downloaded Amiri Arabic font TTF
- Added to `assets/fonts/Amiri-Regular.ttf`
- Updated `pubspec.yaml` to include fonts asset path
- Load font in PDF generator and use for Arabic text fields with RTL direction

#### 2. Face Photo Added
- Extract face image from `contractData.faceImageBase64`
- Decode base64 to bytes and create `pw.MemoryImage`
- Display alongside client info in a bordered container
- Show placeholder if no photo available

#### 3. Offer Section Fixed
- Added fallback values for all offer properties
- Added debug logging to verify data flow
- Used `pw.MultiPage` instead of `pw.Page` to handle overflow

#### 4. Signature Rendering Fixed
- Added proper container with padding
- Verified signature bytes are passed through workflow

**Files Modified:**

| File | Changes |
|------|---------|
| `mobile/assets/fonts/Amiri-Regular.ttf` | NEW - Arabic font file |
| `mobile/pubspec.yaml` | Added `- assets/fonts/` to assets |
| `mobile/lib/services/pdf_generator_service.dart` | Complete rewrite with all fixes |

**Key Code Changes:**

```dart
// Load Arabic font
final arabicFontData = await rootBundle.load('assets/fonts/Amiri-Regular.ttf');
final arabicFont = pw.Font.ttf(arabicFontData);

// Use for Arabic text with RTL
pw.Text(
  arabicText,
  style: pw.TextStyle(font: arabicFont),
  textDirection: pw.TextDirection.rtl,
)

// Load face image from base64
if (contractData.faceImageBase64 != null) {
  final faceBytes = base64Decode(contractData.faceImageBase64!);
  faceImage = pw.MemoryImage(faceBytes);
}

// Use MultiPage for overflow handling
pdf.addPage(
  pw.MultiPage(
    pageFormat: PdfPageFormat.a4,
    build: (context) => [...widgets...],
  ),
);
```

## Changes Made (2025-12-31) - Fix Contract Creation Error

### Issue
Contract creation from mobile app failed with "Erreur lors de la creation du contrat" error, while "Mes Ventes" history page worked correctly.

### Root Cause
The `customer_photo` field mismatch:
- Mobile app sent `customer_photo` as a base64 string
- Django model defined `customer_photo` as `ImageField` (expects file, not string)
- `ImageField` cannot accept base64 strings directly

### Solution

**Backend Changes (`backend/apps/contracts/serializers.py`):**
- Added `customer_photo_base64` field (CharField, write_only)
- In `create()` method: decode base64 and convert to file using `ContentFile`
- Changed field from `customer_photo` to `customer_photo_base64`

```python
customer_photo_base64 = serializers.CharField(
    write_only=True, required=False, allow_blank=True, allow_null=True
)

def create(self, validated_data):
    photo_base64 = validated_data.pop('customer_photo_base64', None)
    if photo_base64:
        image_data = base64.b64decode(photo_base64)
        validated_data['customer_photo'] = ContentFile(
            image_data,
            name=f"photo_{nin}.jpg"
        )
```

**Mobile Changes (`lib/services/api_service.dart`):**
- Changed field name from `customer_photo` to `customer_photo_base64`
